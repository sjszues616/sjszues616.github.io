创建以下目录

  • backup 备份专用目录

  • code 代码专门目录

  • script 脚本专门目录

  • server 服务专用目录

  • soft 软件专用目录

  • virtual 虚拟环境目录

    mkdir /data/{backup,code…}

创建虚拟环境

  • 安装虚拟环境管理软件

    apt-get install python-virtualenv -y

    • 检验安装是否成功 

      which virtualenv

    • 检查当前系统的python版本

      ls /usr/bin/ |grep python

  • 创建虚拟环境

    • 进入项目虚拟环境

      cd /data/virtual

    • 使用指定的python版本创建虚拟环境

      virtualenv -p /usr/bin/python2.7 venv
# tree -L 2 venv 显示两层结构
tree -L 2 venv

  • 进入虚拟环境

    source/. venv/bin/activate
#或者
workon venv
#退出虚拟环境
deactivate
#删除虚拟环境
rm -fr venv

  • 安装django

    apt-get install lrzsz

    rz可以图形化将本地包传输到目标服务器

    sz可以图形化将目标服务器包传输到本地

    • 先进入虚拟环境 source ./venv/bin/activate

    • 按readme.md查看如何安装,并进行安装

    • 源码安装pcre ./configure | make | make install

    • ldd 可执行文件 可以查看可执行文件的依赖库是否都有

    • 软链接的是有 ln -s 文件名 软链接目标地址文件名

    • nginx 常用操作

      • nginx -t 测试
      • nginx 启动
      • nginx -s stop 关闭
      • nginx -s reloas 重载
      • 在配置文件中 proxy_pass 反向代理

      grep -v ‘#’ 文件名 查找没有#的内容 -v 取反

      egrep -v “#|^$” 文件名 支持正则的查找方式
